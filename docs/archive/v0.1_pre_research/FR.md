# WebEnvoy 功能需求清单 (FR List)

> 版本：v0.1  
> 日期：2026年03月12日  
> 状态：草稿  
> 关联文档：[PRD](./PRD.md)

---

## 优先级定义

| 等级 | 含义 | 判断标准 |
|---|---|---|
| **P0** | 必须 | 没有这个功能，系统无法运行 |
| **P1** | 核心 | 没有这个功能，系统没有使用价值 |
| **P2** | 重要 | 没有这个功能，系统能用但体验差或不稳定 |
| **P3** | 增强 | 有了更好，没有不影响核心流程 |

## 一、环境与生命周期管理

| ID | 需求 | 优先级 | 说明 |
|---|---|---|---|
| ENV-01 | 初始化工作目录 | P0 | 创建存放浏览器配置、数据库、插件等资源的本地目录结构 |
| ENV-02 | 启动浏览器（指定配置空间） | P0 | 以独立配置空间启动 Chrome，自动加载内置插件，建立与插件的通信连接 |
| ENV-03 | 关闭浏览器 | P0 | 完全终止浏览器进程，释放资源，不留残余进程 |
| ENV-04 | 平台登录初始化 | P0 | 打开可见浏览器窗口，引导用户手动完成目标平台的登录，并持久化登录状态 |
| ENV-05 | 多配置空间管理 | P1 | 支持创建、列出、删除多个独立的配置空间（用于多账号隔离） |
| ENV-06 | 查看当前浏览器状态 | P2 | 返回浏览器是否在运行、当前使用的配置空间名称、打开的标签页数量等信息 |
| ENV-07 | 浏览器崩溃自动检测 | P2 | 监测浏览器进程存活状态，崩溃时主动通知 AI |
| ENV-08 | 执行层策略切换 | P1 | 支持三种执行引擎：安全模式（OS 级输入模拟，窗口须可见）、标准模式（插件执行，默认）、效率模式（CDP 协议，速度最快但有检测风险）。用户可按配置空间或全局设置 |
| ENV-09 | 通信层通道切换 | P1 | 支持两种底层通信通道：Native Messaging（默认，最安全稳定）和 WebSocket（开发调试友好） |
| ENV-10 | 启动时校验登录状态 | P1 | 浏览器启动后自动检测已保存的平台登录状态是否仍有效（Cookie/Session 未过期）。有效时返回"已登录"及账号标识；失效时返回"登录已失效"提示但不阻塞启动 |
| ENV-11 | 列出配置空间详情 | P1 | 返回所有配置空间的列表，每项包含：配置名称、关联平台、创建时间、最近使用时间，便于 AI 选择目标配置 |

## 二、CLI 与插件通信

| ID | 需求 | 优先级 | 说明 |
|---|---|---|---|
| COM-01 | CLI 与插件建立双向通信通道 | P0 | 浏览器启动后，插件应主动与 CLI 建立连接，支持双向消息传递。插件需声明的最小 Chrome 权限集（tabs、webRequest、nativeMessaging、storage 等）应在初始化文档中明确列出，启动时验证权限完整性 |
| COM-02 | 通信通道断线重连 | P2 | 插件后台进程被浏览器回收后（MV3 Service Worker 限制），应能自动重新建立连接 |
| COM-03 | 心跳保活 | P2 | CLI 与插件之间定期检测连接存活，及时发现断连 |
| COM-04 | 通信通道自动降级 | P2 | 首选通道（如 Native Messaging）建立失败时，自动尝试备选通道（如 WebSocket）。降级成功时通知 AI 当前通道及降级原因；所有通道均失败时返回错误信息和排查建议 |

## 三、页面导航与观察

| ID | 需求 | 优先级 | 说明 |
|---|---|---|---|
| NAV-01 | 打开指定网址 | P0 | 在当前标签页或新标签页中打开指定 URL，等待页面加载完成后返回确认 |
| NAV-02 | 获取当前页面信息 | P1 | 返回当前页面的标题、URL、加载状态 |
| NAV-03 | 获取页面可交互元素列表 | P1 | 返回当前可见区域内所有可点击、可输入的元素及其语义描述 |
| NAV-04 | 获取页面正文内容（Markdown） | P1 | 将当前页面的核心正文内容提取为精简的 Markdown 格式文本 |
| NAV-05 | 获取页面截图 | P2 | 截取当前可见区域的截图并保存到本地，返回文件路径。支持可选参数截取整个长页面的全尺寸截图 |
| NAV-06 | 管理标签页 | P2 | 支持新建、切换、关闭标签页。切换回原标签页时，页面的滚动位置和输入状态应保持不变 |
| NAV-07 | 获取页面无障碍树 (感知层) | P1 | 提取当前页面的精简无障碍树 (Accessibility Tree)，信息量压缩至原始页面的 5% 以内。优先集成成熟的开源 Provider（如 PinchTab），并支持架构上的平滑替换 |
| NAV-08 | 登录墙检测 | P2 | 通用网页读取时，自动检测页面是否为登录页/登录墙。检测到时返回"页面需要登录"提示及平台标识（如可识别），而非将登录页内容当作目标内容返回 |

## 四、页面交互操作

| ID | 需求 | 优先级 | 说明 |
|---|---|---|---|
| ACT-01 | 点击指定元素 | P0 | 根据元素标识定位元素，滚动至可见后模拟点击。元素不存在时返回错误。标识匹配到多个元素时不自动选择，而是返回各匹配元素的简要描述（位置、文本），由 AI 使用更精确的标识重新发起 |
| ACT-02 | 在输入框中填写文字 | P0 | 定位目标输入框，模拟逐字输入并触发页面框架的状态更新。应自动识别输入区域类型（标准 input/textarea vs contenteditable 富文本编辑器），根据类型采用相应输入策略 |
| ACT-03 | 页面滚动 | P1 | 支持按方向、距离或"滚动到指定元素"进行页面滚动 |
| ACT-04 | 上传本地文件 | P1 | 将本地文件传递给页面的文件上传组件。自动检测上传组件类型：标准 `<input type="file">` 通过注入文件完成，拖拽上传区域通过模拟拖放事件（dragenter/dragover/drop）完成。上传后等待页面显示预览或进度后再返回确认 |
| ACT-05 | 等待指定元素出现 | P1 | 在指定时间内轮询等待某个元素出现在页面中，超时返回失败 |
| ACT-06 | 输入拟真：随机节奏 | P1 | 输入文字时字符间有随机时间间隔（快慢不一），偶尔有较长停顿（模拟思考） |
| ACT-07 | 输入拟真：模拟纠错 | P2 | 输入过程中偶尔出现删除一两个字符后重新输入的行为（模拟打错字） |
| ACT-08 | 点击拟真：坐标偏移 | P1 | 点击位置不是元素的精确中心，而是在合理范围内随机偏移 |
| ACT-09 | 点击拟真：鼠标轨迹 | P2 | 点击前鼠标从当前位置沿自然曲线移动到目标位置（非瞬移），安全模式下为 OS 级真实鼠标移动 |
| ACT-10 | 鼠标悬停 | P3 | 将鼠标移动到指定元素上方，触发 hover 效果 |

## 五、数据采集与拦截

| ID | 需求 | 优先级 | 说明 |
|---|---|---|---|
| DAT-01 | 拦截指定 URL 模式的网络响应数据 | P1 | 插件在页面上下文中拦截匹配特定模式的接口请求，捕获其 JSON 响应数据 |
| DAT-02 | 采集到的数据自动存入数据库 | P1 | CLI 接收到插件传回的数据后，按统一格式写入本地 SQLite 数据库 |
| DAT-03 | 存储时自动标记来源信息 | P1 | 每条数据自动记录来源平台、数据类型、采集时间、批次号 |
| DAT-04 | 基于业务 ID 的数据去重 | P2 | 同一平台的同一内容（如同一篇笔记）不重复插入，而是更新已有记录 |
| DAT-05 | 采集入库确认反馈 | P1 | 每次采集完成后，摘要中应包含实际采集条数、成功入库条数、去重跳过条数的比对信息，以及每条数据的数据库记录标识 |

| MED-04 | 媒体文件保存目录管理 | P2 | 支持按配置空间、平台、日期等维度组织保存的媒体文件 |

## 七、数据查询与管理

| ID | 需求 | 优先级 | 说明 |
|---|---|---|---|
| QRY-01 | 通过条件查询数据库中的数据 | P1 | AI 可通过 SQL 或简化的查询语法从数据库中检索数据 |
| QRY-02 | 查询操作执行历史 | P2 | 查询某个目标（如某用户）是否已执行过特定操作（用于去重和防重复） |
| QRY-03 | 按时间范围清理过期数据 | P2 | 删除指定时间之前的原始采集数据，释放磁盘空间。用户标记为"保留"的数据不应被清理 |
| QRY-04 | 数据导出 | P3 | 将查询结果导出为 CSV 或 JSON 文件 |
| QRY-05 | 标记数据为保留 | P2 | 对指定数据记录标记为"保留"，防止被自动清理；保留标记可随时取消 |

## 八、信息披露控制

| ID | 需求 | 优先级 | 说明 |
|---|---|---|---|
| DSC-01 | 采集结果返回摘要而非全量 | P0 | 任何产生大量数据的命令，终端输出应只包含摘要（条数、预览），完整数据存入数据库 |
| DSC-02 | 查询结果分页加载 | P1 | 查询结果支持分页返回，避免单次返回过多数据，防止 Token 爆炸 |
| DSC-03 | 按需获取指定字段 | P1 | AI 可指定只返回哪些字段，避免返回无关信息，提高信噪比 |
| DSC-04 | 错误信息结构化返回 | P1 | 所有错误应以统一格式返回，包含错误类型和可理解的描述 |
| DSC-05 | 操作结果附带状态反馈 | P1 | 每个命令的返回都应包含明确的成功/失败/等待状态 |
| DSC-06 | 写操作结果结构化返回 | P1 | 发布/发送类写操作成功时，返回应包含平台侧的关键标识信息（如笔记 ID/URL、发布时间、审核状态），便于 AI 后续跟踪和引用 |

## 九、异常处理与容错

| ID | 需求 | 优先级 | 说明 |
|---|---|---|---|
| ERR-01 | 验证码检测与任务暂停 | P1 | 检测到验证码时暂停任务，通知 AI 需要人工介入 |
| ERR-02 | 验证码通过后恢复任务 | P2 | 人工完成验证后，系统能够恢复暂停的任务继续执行 |
| ERR-03 | 页面加载异常处理 | P1 | 页面超时、HTTP 错误（4xx/5xx）或被平台拦截时返回明确的错误类型，不无限等待 |
| ERR-04 | 元素查找超时处理 | P1 | 等待元素超时后返回明确错误 |
| ERR-05 | 操作频率限流 | P2 | 内置各平台的安全操作间隔，超频时自动延迟而非拒绝，并通知 AI 当前正在冷却等待，达到安全间隔后自动恢复执行 |
| ERR-06 | 操作执行记录 | P2 | 将每次写操作（发消息、发帖等）记录到数据库，用于断点续传和审计 |
| ERR-07 | 插件连接中断检测与恢复 | P2 | 浏览器仍在运行但插件 Service Worker 被回收时，自动尝试重连；重连期间命令排队等待；重连失败则通知 AI。插件被用户手动禁用/卸载时通知 AI 需重启浏览器 |
| ERR-08 | 网络连接中断检测 | P2 | 因本机网络问题导致页面加载失败时，返回"网络不可用"的明确错误类型（区别于页面自身的 4xx/5xx 错误），网络恢复后系统不应残留异常状态 |
| ERR-09 | 登录会话过期检测 | P1 | 操作过程中检测到平台返回"需要登录"的响应（跳转登录页或接口返回 401）时，暂停操作并返回"登录状态已过期"的错误，提示用户重新执行登录流程 |
| ERR-10 | 账号封禁/限制检测 | P2 | 检测到平台返回账号受限、封禁等响应时，立即停止操作序列并返回错误及平台提示原文。不自动重试写操作以避免加重封禁。封禁场景下暂停该配置空间的写操作并通知用户，由用户判断封禁性质后手动解除暂停 |
| ERR-11 | 专用工具降级兜底 | P2 | 专用工具因平台页面结构变更而失效时（未拦截到预期请求或数据格式不匹配），返回明确错误并提示 AI 可使用兜底工具替代。附带诊断细节便于排查 |
| ERR-12 | 数据库不可用处理 | P2 | 数据库因文件损坏、被锁定或磁盘空间不足而不可写入时，返回错误类型和具体原因。读操作类命令的核心结果（如网页内容）仍应返回给 AI，仅标注"数据未入库"。磁盘空间不足时建议执行数据清理 |
| ERR-13 | CLI 退出后资源清理 | P2 | CLI 意外退出后浏览器继续运行不自动关闭。CLI 重启时检测到残留浏览器应尝试重新连接（而非启动新实例），重连失败则提示用户手动关闭残留进程 |

## 十一、反检测措施

| ID | 需求 | 优先级 | 说明 |
|---|---|---|---|
| STL-01 | 清除基础自动化指纹 | P1 | 启动时自动清除浏览器中常见的自动化标识（如 webdriver 属性） |
| STL-02 | 配置空间指纹一致性 | P2 | 同一配置空间每次启动时保持一致的浏览器指纹特征，不每次随机变化 |
| STL-03 | 执行层：OS 级输入引擎 (安全模式) | P2 | 通过操作系统层面模拟真实鼠标移动和键盘输入（macOS CGEvent / Windows SendInput），所有交互事件完全真实。需浏览器窗口可见 |
| STL-04 | 执行层：CDP 引擎 (效率模式) | P2 | 通过浏览器调试协议执行操作，速度最快，适用于风控宽松的平台或用户接受风险的场景 |

## 十二、边界情况处理

| ID | 需求 | 优先级 | 说明 |
|---|---|---|---|
| EDG-01 | 工作目录重复初始化保护 | P2 | 工作目录已存在时再次执行初始化命令，应跳过并提示"已存在"，不覆盖或删除任何已有数据 |
| EDG-02 | 浏览器重复启动保护 | P2 | 对已在运行的配置空间再次发出启动命令时，不启动第二个实例，返回"已在运行"并附带当前状态。对不同配置空间则允许启动独立实例，命令通过配置空间名称路由 |
| EDG-03 | 特殊字符输入处理 | P1 | 正确传递 emoji、换行符、零宽字符等特殊字符。emoji 等多字节字符应作为整体输入。输入超出平台字数限制时检测并反馈超限状态（当前字数/上限字数），由 AI 决定后续处理 |
| EDG-04 | 元素标识歧义处理 | P1 | 元素标识匹配到多个可见元素时返回"标识不唯一"错误，附带匹配数量及各元素简要描述（位置、文本片段），AI 可据此用更精确的标识重试 |
| EDG-05 | 写操作目标不可达处理 | P2 | 私信目标用户已注销/拉黑/关闭私信、评论目标笔记已删除/关闭评论区时，返回"目标不可达"错误类型及平台提示原文 |
| EDG-06 | 上传文件前置校验 | P2 | 上传前校验文件是否存在，不存在则直接报错不发送请求。上传后文件格式不被平台接受或超出大小限制时，检测页面反馈并返回具体错误 |
| EDG-07 | iframe 内元素处理 | P2 | 获取可交互元素列表时，同源 iframe 内的元素应被包含并标注所属 iframe；跨域 iframe 内的元素不可访问，应在返回中标注"存在跨域 iframe，内部元素不可操作" |
| EDG-08 | 媒体资源不可用处理 | P2 | 缓存中未找到目标媒体资源时返回提示并建议刷新页面后重试。资源因防盗链返回 403 或空内容时返回明确错误并提示优先使用缓存提取方式 |
| EDG-09 | 配置空间命名校验 | P2 | 创建配置空间时校验名称合法性：禁止文件系统特殊字符（/ \ : * ? " < > |）、长度限 1-64 字符、不允许空字符串或纯空格、不允许与已有配置空间重名。校验失败时返回具体原因和命名规则说明 |
| EDG-10 | 超大页面元素上限 | P2 | 页面可交互元素超过 500 个时，NAV-03 和 NAV-07 默认返回前 200 个元素并标注总数。支持通过参数指定区域（如仅当前视口）或偏移量分批获取后续元素 |

## 十三、空状态处理

| ID | 需求 | 优先级 | 说明 |
|---|---|---|---|
| EMP-01 | 数据库空/无匹配查询返回 | P1 | 数据库为空或查询条件无匹配时，返回统一空结果格式 `{ total: 0, items: [] }` 及提示信息，不返回错误。无匹配时附带该数据类型的总量信息帮助 AI 判断是条件过严还是确实无数据 |
| EMP-02 | 搜索零结果处理 | P1 | 平台搜索返回零结果时，返回空结果格式且不向数据库写入记录。如平台页面显示搜索建议或提示信息应一并返回。请求数量超过实际结果时如实返回并标注"已到达结果末尾" |
| EMP-03 | 用户主页无笔记 | P2 | 目标用户存在但未发布过笔记时，正常返回用户基本资料（昵称、简介、粉丝数等），笔记列表部分返回 `{ notes_count: 0, notes: [] }`，用户资料仍入库 |
| EMP-04 | 私信列表为空 | P2 | 当前账号无私信记录时返回 `{ total: 0, conversations: [] }` 及提示，不返回错误，不向数据库写入空记录 |
| EMP-05 | 页面无可交互元素 | P1 | 纯静态展示页面（无按钮、无输入框）执行检视命令时返回 `{ total: 0, elements: [] }` 及提示，建议 AI 改用 NAV-04（正文提取）或 NAV-07（语义树）理解页面 |
| EMP-06 | 导出无匹配数据 | P2 | 导出时查询条件无匹配记录，返回提示"无匹配数据，未生成文件"，不创建空文件，附带可导出数据的总量信息 |
| EMP-07 | 缓存中无目标媒体 | P2 | 浏览器缓存中不存在目标图片/视频文件时，返回"缓存未命中"提示。图片建议先滚动页面使其完整加载后重试；视频建议先播放使其缓存后重试。不自动发起额外网络请求（遵守被动观察原则） |
| EMP-08 | 配置空间列表为空 | P2 | 工作环境已初始化但未创建任何配置空间时，列出命令返回 `{ total: 0, profiles: [] }` 及提示，建议先执行平台登录命令创建配置空间 |

## 十四、数据一致性保障

| ID | 需求 | 优先级 | 说明 |
|---|---|---|---|
| CON-01 | 采集中断时保留已入库数据 | P1 | 批量采集中断时，已成功写入的数据保留不回滚，批次标记为"未完成"并记录中断位置和原因。重新执行相同任务时告知上次进度 |
| CON-02 | 媒体文件与数据库记录同步 | P2 | 文件先保存再写记录；文件保存成功但记录失败时保留文件并警告；文件保存失败则不创建记录。查询时如文件缺失应标注状态而非报错 |
| CON-03 | 写操作结果优先于记录 | P1 | 写操作的实际执行结果（成功/失败）是最终真相，操作记录写入失败不影响结果返回，附带"记录未入库"警告。失败的操作也应准确记录失败状态和原因，不遗漏。记录写入失败时尝试延迟补写 |
| CON-04 | 并发写入去重安全 | P2 | 多条数据因并发几乎同时到达时，基于业务 ID 的去重应保证原子性（检查+插入为一个事务），不产生重复记录且不锁死数据库。后到达的数据触发更新而非报错 |
| CON-05 | 数据清理级联删除 | P2 | 清理数据库记录时同步删除关联的本地媒体文件。汇报释放的记录数、文件数和磁盘空间。"保留"标记的数据及关联文件不受影响 |
| CON-06 | 配置空间删除级联清理 | P2 | 删除配置空间时完整清理关联的浏览器配置、数据库记录和媒体文件，不影响其他配置空间。删除前提示影响范围（记录数、文件数、总大小）并要求确认 |

## 十五、故障恢复

| ID | 需求 | 优先级 | 说明 |
|---|---|---|---|
| RCV-01 | 批量采集断点续传 | P2 | 未完成的采集批次记录断点信息（已采集数量、搜索参数）。重复执行相同任务时提示 AI 是否从断点继续或重新开始。选择重新开始时将旧批次标记为"已废弃" |
| RCV-02 | 数据库自动备份 | P2 | 满足条件时（每日首次启动 / 累计写入超过 1000 次）自动备份数据库文件。保留最近 N 个备份（默认 3），自动清理旧备份，备份不阻塞正常操作 |
| RCV-03 | 数据库损坏检测与恢复引导 | P2 | 访问数据库时自动执行 integrity check，损坏时返回明确错误并停止写入。有备份时提示恢复（含备份日期），无备份时提示重建（保留配置空间和登录状态，数据丢失） |
| RCV-04 | CLI 退出后重连浏览器 | P2 | CLI 重启时检测残留浏览器进程并尝试与插件重连。重连成功则返回"已重新接管"及当前浏览器状态，失败则提示手动关闭残留进程后重启 |
| RCV-05 | 页面无响应检测 | P2 | 命令发出后页面超时（默认 30s）无响应时返回"页面无响应"错误，建议 AI 尝试刷新页面、关闭标签页或重启浏览器等恢复策略，不无限等待 |
| RCV-06 | 限流冷却状态持久化 | P3 | 冷却截止时间写入持久化存储，CLI 重启后读取并恢复冷却计时。未到期则继续等待，已过期则正常放行 |

## 十六、状态管理与并发控制

| ID | 需求 | 优先级 | 说明 |
|---|---|---|---|
| SCC-01 | 浏览器生命周期状态机 | P0 | 维护每个浏览器实例的完整状态（未启动/启动中/就绪/执行中/已暂停/关闭中/异常断开），严格定义合法转换路径。操作命令执行前须校验当前状态，不合法时返回拒绝原因和当前状态 |
| SCC-02 | 浏览器状态查询 | P1 | AI 可随时查询浏览器当前状态及上下文信息：执行中附带命令名称和耗时，已暂停附带暂停原因，异常断开附带断开时间和最后已知状态 |
| SCC-03 | 通信连接状态管理 | P1 | 维护 CLI-插件通信连接的状态（未连接/连接中/已连接/重连中/已断开）。重连期间操作命令排队等待（上限 5 条），30 秒内重连成功则排队命令按序执行，超时则排队命令全部返回错误。状态变化同步反映在浏览器状态查询中 |
| SCC-04 | 登录状态持久化跟踪 | P1 | 每个配置空间维护独立的登录状态（未登录/已登录/已失效）并持久化到元数据中。状态变更时主动通知 AI。操作中检测到登录失效时浏览器状态从"执行中"转为"已暂停"（与 ERR-09 协同） |
| SCC-05 | 命令串行执行 | P0 | 同一浏览器实例的操作命令（点击/输入/导航/采集等）严格串行，执行中收到新操作命令时立即拒绝并返回冲突信息（正在执行的命令名称和耗时），不排队 |
| SCC-06 | 查询命令不阻塞 | P1 | 数据库查询、状态查询、数据导出、配置空间列表等非浏览器操作命令不受串行限制，可在浏览器"执行中"状态下并行执行，不影响当前操作 |
| SCC-07 | 配置空间独占锁 | P1 | 通过锁文件实现互斥，同一配置空间同时只能被一个 CLI 进程使用。被占用时返回错误含占用进程 PID 和启动时间。占用进程已崩溃（PID 不存在）时自动清理残留锁。CLI 退出时自动释放锁 |
| SCC-08 | 多浏览器实例命令路由 | P2 | 多个实例同时运行时，操作命令须指定目标配置空间，系统据此路由。仅一个实例运行时可自动路由。目标实例未启动时返回错误并列出运行中实例。CLI 应能列出所有运行中实例及其状态 |
| SCC-09 | 数据库读写并发安全 | P1 | SQLite 启用 WAL 模式，保证读写并发时的数据安全（多个读操作与单个写操作可同时进行）。写入过程中的查询应能正常执行并返回一致数据（不读到半写记录），写入不被查询阻塞 |
| SCC-10 | 执行策略热切换 | P2 | 浏览器运行期间可切换执行策略：就绪状态下立即生效，执行中状态下等待当前命令完成后生效。切换到安全模式时须检查窗口可见性，不满足则拒绝并保持原策略。切换无需重启浏览器 |

## 十七、性能约束

| ID | 需求 | 优先级 | 说明 |
|---|---|---|---|
| PER-01 | 命令响应延迟要求 | P1 | 纯本地命令（数据库查询、状态查看）1 秒内返回；页面操作命令 30 秒内返回结果或超时错误；数据采集命令 5 秒内返回首条中间反馈。超时命令支持 AI 主动取消 |
| PER-02 | 无障碍树提取超时控制 | P2 | 复杂页面的无障碍树提取应在 5 秒内完成。超时时返回已提取的部分结果并标注不完整。提取过程不应导致页面卡顿或浏览器无响应 |
| PER-03 | 数据库索引与查询性能 | P2 | 为高频查询字段（平台、数据类型、业务 ID、采集时间）创建索引。10 万条记录下索引查询 1 秒内返回，复杂查询不超过 5 秒 |
| PER-04 | Native Messaging 消息大小控制 | P2 | 首选策略是通过压缩、降质、摘要等方式将消息体控制在 Chrome Native Messaging 的 1MB 限制以内；确实无法压缩到 1MB 以下时，作为兜底方案自动将大消息拆分为多条分片传输并在接收端重组。WebSocket 通道单条消息上限 10MB |
| PER-05 | 磁盘空间监控与告警 | P2 | 可用空间低于 500MB 时附带警告并建议清理；低于 100MB 时停止媒体文件采集（数据库写入仍允许）。告警信息包含工作目录的磁盘占用明细（数据库大小、媒体文件大小） |
| PER-06 | 长文本拟真输入性能 | P2 | 拟真输入超过 3 秒时返回中间状态及预计完成时间。支持可选参数关闭拟真模式以加速输入。拟真模式下实际耗时不超过字数 × 200ms |

## 十八、安全保护

| ID | 需求 | 优先级 | 说明 |
|---|---|---|---|
| SEC-01 | CLI-插件通信认证 | P1 | Native Messaging 通过 Chrome allowed_origins 限制连接来源；WebSocket 通过一次性令牌验证。未认证连接立即拒绝，不返回系统信息 |
| SEC-02 | 查询注入防护 | P1 | 所有数据库操作使用参数化查询（Prepared Statement）。SQL 模式仅允许 SELECT 语句，禁止 INSERT/UPDATE/DELETE/DROP 等写操作。单次查询结果硬上限 1000 条 |
| SEC-03 | 批量写操作安全上限 | P2 | 按操作类型设置单日上限（私信 50 条/评论 100 条/发布 20 篇，每配置空间独立计数，目的是保护单个账号不触发平台风控），达到上限前预警，达到后拒绝同类操作。上限可通过配置文件调整（需手动确认风险），每自然日零点重置 |
| SEC-04 | 工作目录文件权限 | P2 | 工作目录和配置空间目录权限设为仅当前用户可读写（Unix 700/600）。数据库文件同理。临时文件（截图、导出）由 AI 决定保留或清理 |
| SEC-05 | Content Script 执行隔离 | P1 | 插件 Content Script 运行在 Chrome Isolated World 中，与页面 JS 隔离。内部消息传递包含不可预测的标识以防伪造。不向页面暴露全局变量或 DOM 修改 |
| SEC-06 | 敏感数据自动清理 | P2 | 采集的个人信息超过保留期限（默认 90 天）且未标记"保留"时自动删除，关联媒体文件同步清理。初始化时提示用户数据合规注意事项 |
| SEC-07 | 操作审计日志 | P2 | 所有写操作记录操作类型、目标标识、执行时间、使用的配置空间、执行结果。审计日志独立存储，不受数据清理影响，保留 180 天后自动清理。AI 可查询但不可修改或删除 |
| SEC-08 | macOS 辅助功能权限引导 | P2 | 安全模式（OS 级输入模拟）在 macOS 上首次使用时检测辅助功能权限状态。未授权时返回明确的授权引导步骤（而非通用权限错误），权限授予后自动验证可用性。Windows/Linux 如需类似权限也应提供对应引导 |
