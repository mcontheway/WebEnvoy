# FR-0005 小红书 Spike A：核心读链路 API 与签名确认

## 背景

Phase 1 的第 4 个必需件是“首个平台核心读闭环”。`#145` 的职责不是直接交付完整平台适配器，而是先把小红书读链路里最关键且最不确定的三类输入固定下来：

- 核心读取 API 端点与请求形态
- 页面内签名函数调用路径
- 关键追踪字段与生命周期

没有这层 Spike 证据，后续 L3 读能力会退化成猜测式实现，无法进入稳定迭代。

## 目标

1. 确认小红书核心读链路最小端点集（搜索、详情、用户主页）及其请求/响应关键字段。
2. 确认页面签名函数的最小调用路径，证明可在浏览器内执行链路中稳定触发。
3. 形成可复用的“侦察证据包”与结构化结论，作为后续 L3 读适配 FR 的确定性输入。
4. 与 FR-0001/0002/0003 的运行时契约对齐，不引入第二套命令或消息壳。

## 非目标

- 不实现完整小红书读适配器。
- 不实现发布、互动、下载等写入能力。
- 不扩张到多平台并行侦察框架。
- 不在本 FR 内引入新的 SQLite schema 变更。

## 功能需求

### 1. 核心端点侦察范围冻结

- 必须覆盖三类读取场景：搜索列表、笔记详情、用户主页。
- 每类场景至少沉淀一个可复现端点与关键请求字段。
- 必须记录请求方法、路径、最小必需 headers、关键 query/body 字段。

### 2. 签名链路最小可证实

- 必须确认页面签名函数的调用入口、最小输入、输出结构。
- 必须给出“可复现调用”证据，不接受仅靠静态猜测。
- 如果存在版本漂移风险，必须标注检测点和失效症状。

### 3. 关键追踪字段生命周期

- 对请求中影响风控或路由的关键字段，必须标注来源（页面状态/运行时生成/固定值）。
- 必须标注字段生命周期：会话级稳定、请求级变化、页面刷新后变化。
- 必须明确哪些字段属于后续适配器强依赖，哪些仅为可选增强。

### 4. 结构化侦察输出

- Spike 输出必须是结构化结论，而不是散乱笔记。
- 输出应能直接被后续 FR 引用，至少包含：
  - 端点清单
  - 签名链路结论
  - 关键字段矩阵
  - 失效信号与验证步骤

### 5. 与现有运行时契约对齐

- 侦察执行必须依赖 FR-0002/0003 已定义的浏览器内链路与会话边界。
- 不允许定义与 FR-0001 CLI 契约冲突的新外层输入输出模型。

## GWT 验收场景

### 场景 1：核心端点可以被稳定识别

Given Native Messaging 与最小会话链路可用  
When 对小红书执行搜索、详情、主页三类读取侦察  
Then 每类场景都能给出至少一个已验证端点  
And 每个端点都包含方法、路径与关键请求字段  

### 场景 2：签名调用路径可复现

Given 页面处于登录态并可访问对应功能页  
When 在浏览器内上下文执行签名链路验证  
Then 能得到可复现的签名调用入口和输出  
And 失败时能区分“调用入口失效”和“参数不匹配”  

### 场景 3：关键字段生命周期被明确标注

Given 已捕获核心读链路请求样本  
When 对关键追踪字段做多次会话/请求对比  
Then 每个关键字段都有来源与生命周期归类  
And 能明确后续适配器的强依赖字段集合  

### 场景 4：输出可直接服务后续 FR

Given Spike 侦察已经完成  
When 评审者检查输出文档  
Then 能直接提取 L3 读适配所需的端点和签名结论  
And 不需要二次逆向才能进入实现 planning  

## 异常与边界场景

### 1. 网络与并发

- 侦察期间出现短时超时或 429 时，必须记录失败证据，不得用成功样本覆盖失败事实。
- 同一端点多次请求返回不一致时，必须标注触发条件和复现实验。

### 2. 空与极值

- 搜索空结果、无效 note_id、无作品用户主页都要有样本与结论。
- 超长关键词或特殊字符导致签名输入变化时，必须给出边界说明。

### 3. 权限与状态异常

- 未登录、会话过期、风控拦截（验证码/滑块）必须单独记录，不得混写为普通请求失败。
- 账号权限不足导致端点行为变化时，必须标记为环境前置限制。

### 4. 外部依赖故障

- 页面脚本更新导致签名入口变化时，必须提供最小排查路径。
- Extension 注入失败或 content-script 不可达时，必须明确这属于运行时链路问题，而非平台接口问题。

## 验收标准

1. 三类核心读场景均有“已验证端点 + 关键字段”结论。
2. 签名函数调用路径可复现，并有失效信号说明。
3. 关键追踪字段的来源与生命周期矩阵完整。
4. 侦察结论可直接作为后续 L3 读适配 FR 输入。
5. 文档中没有引入与 FR-0001/0002/0003 冲突的契约。
