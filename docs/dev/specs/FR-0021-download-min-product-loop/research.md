## 结论

1. 下载是 Phase 2 统一能力面之一，不能继续只停留在 follow-up issue。
2. 下载能力必须复用 `FR-0017` 的统一 descriptor，而不是另起一套下载协议。
3. 下载能力可以进入 `FR-0018` 的普通 `read|download` trust 域，因此现在需要先冻结请求、结果与 artifact 引用边界。
4. 最小产品闭环应先收口单文件/单媒体资产，不扩张到完整下载管理器。

- 若继续停留在 issue 层，后续实现容易把落盘、artifact、来源 URL 和候选能力 handoff 混在一起。
