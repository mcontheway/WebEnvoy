# FR-0012 风险与回滚

## 风险 1：Layer 1 实现绕过既有门禁与状态机

- 触发条件：实现把 profile patch 预加载、失败重试或后台 warmup 做成绕过 `FR-0010/0011` 的隐式 live 通道。
- 影响：Sprint 2/3 已冻结的审批、审计、状态机前置失效。
- 缓解：
  - 明确 FR-0012 只消费 `FR-0010/0011` 的既有对象。
  - Layer 1 失败或失配必须通过既有审计/错误链路回传。
- 回滚：
  - 阻断该实现 PR。
  - 恢复到未启用 Layer 1 扩展补丁的既有执行路径。

## 风险 2：profile 缺字段时临时随机补值

- 触发条件：实现发现 `audioNoiseSeed`、Battery 或硬件字段缺失后，运行时即时随机生成。
- 影响：指纹跨 run 漂移，比无补丁更容易暴露自动化。
- 缓解：
  - 把 `fingerprint_profile_bundle` 定义为稳定机器边界。
  - 缺字段时返回结构化错误，不允许静默补值继续高风险 live。
- 回滚：
  - 回退到 `dry_run/recon` 或阻断 live。
  - 要求先补齐 profile 元数据后再继续实现验证。

## 风险 3：跨环境迁移被误当成可复用 profile

- 触发条件：在不同 OS、版本或架构环境直接复用同一 profile。
- 影响：Canvas/Audio 基准值变化导致指纹突变，live 风险急剧上升。
- 缓解：
  - 冻结 `os_family`、`os_version`、`arch` 绑定约束。
  - 失配时明确阻断/降级方向。
- 回滚：
  - 停止该 profile 的 live 使用。
  - 在一致环境重新创建或重新播种 profile。

## 风险 4：把 Layer 1 夸大为完整 JS 指纹闭环

- 触发条件：实现或 PR 描述忽略 Worker 线程盲区，宣称“完整补齐 JS 指纹”。
- 影响：后续 live 失败会被误归因，review 也会错误放行。
- 缓解：
  - 在 `spec.md` 中显式保留 Worker 非目标。
  - 将该类过度承诺视为 spec review 阻断项。
- 回滚：
  - 收回误导性描述。
  - 重新按 Layer 1 主线程补全口径审查。

## 风险 5：P0/P1/P2 切片口径漂移

- 触发条件：后续实现 PR 未声明本次只覆盖哪个优先级切片，却使用“完成 #235”之类表述。
- 影响：issue 关闭语义和交付事实不一致，后续补丁无人承接。
- 缓解：
  - 在 FR-0012 中冻结优先级切片。
  - 要求实现 PR 显式声明本次覆盖范围。
- 回滚：
  - 不关闭 `#235`。
  - 将 PR 改回 `Refs #235` 并明确剩余范围。

## Stop-Ship 条件

- FR-0012 spec review 未通过却启动 Layer 1 实现 PR。
- profile 缺字段时系统仍以随机补值继续高风险 live。
- 环境失配未被记录，却仍把 profile 当作稳定指纹 profile 使用。
- 实现 PR 改写 `FR-0010/0011` 的门禁/状态机语义。
- PR 或文档把 Worker 线程问题表述成“本 FR 已解决”。
