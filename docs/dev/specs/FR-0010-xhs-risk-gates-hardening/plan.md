# FR-0010 实施计划

## 实施目标

在不新增平台业务能力的前提下，实现 XHS 读写路径的最小执行硬门禁：读写域分离、目标域/目标页显式确认、默认 `dry_run/recon`、live 升级前置、人工确认与审计记录，并将 `#208/#209` 统一纳入门禁。

## 分阶段拆分

### 阶段 A：门禁契约冻结与接入点梳理

- 产出：
  - 门禁输出对象与字段语义（见 `contracts/risk-gate-execution.md`）
  - 关键执行接入点清单（读路径、写路径候选、模式切换）
- 依赖：`FR-0009` 已合并。

### 阶段 B：执行前门禁实现

- 产出：
  - 读域/写域分离检查
  - 显式目标域/目标页确认检查
  - 默认模式切换为 `dry_run/recon`
  - 高风险 live 默认阻断

### 阶段 C：人工确认与审计收口

- 产出：
  - live 升级审批前置
  - 审计记录落盘与查询契约
  - 失败回退与暂停策略

### 阶段 D：事项联动与门禁验收

- 产出：
  - `#208` 的 live 恢复前置口径
  - `#209` live 扩展受控口径
  - 针对门禁行为的验证证据与回滚演练

## 实现约束

1. 不得在本 FR 中交付新的平台业务能力命令。
2. 不得将门禁逻辑只做文档约定，必须有可执行判定与输出对象。
3. 不得允许高风险 live 在“无审批记录”情况下放行。
4. 不得把读域结论直接应用到写域。
5. 不得修改 `FR-0001` CLI 外层契约语义。

## 测试与验证策略

- 合约层验证：
  - 门禁对象字段完整性、枚举值合法性、回退语义稳定。
- 行为层验证：
  - 未满足前置时默认 `dry_run/recon`；
  - 高风险 live 被阻断；
  - 目标页未确认时阻断；
  - 审批记录缺失时阻断。
- 事项联动验证：
  - `#208/#209` 调用链能消费门禁结论并遵循阻断规则。

## TDD 范围

- 必须 TDD：
  - 门禁判定核心逻辑
  - 审批前置判定逻辑
  - 门禁结果序列化/反序列化与审计记录映射
- 可后置：
  - 低风险辅助日志字段增强
  - 非核心观测字段扩展

## 并行 / 串行关系

- 串行：
  - 阶段 A -> 阶段 B -> 阶段 C -> 阶段 D
  - `FR-0010` spec review 通过前，不进入实现 PR。
- 可并行：
  - 与 Sprint 3 的前置规约研究可以并行，但实现不得越过本 FR 门禁边界。
  - Roadmap 文档重排可并行推进。

## 进入实现前条件

1. `FR-0010` spec review 通过。
2. `contracts/`、`risks.md`、`research.md`、`data-model.md` 完整且可审。
3. 已明确 `#208` 与 `#209` 的门禁消费关系。
4. 已定义“默认非 live”与“live 恢复”判定条件。
5. 已确认回滚策略：任一关键前置失效时立即回退到 `dry_run/recon`。
