# FR-0010 风险与回滚

## 风险 1：门禁只停留在文档层，未形成执行前硬检查

- 等级：critical
- 触发条件：spec review 后直接恢复 live，未实现门禁判定逻辑。
- 影响：`#208/#209` 继续在无硬约束下扩展 live 风险面。
- 缓解：
  - 把门禁实现列为 Sprint 2 必做项。
  - 以自动化测试验证“未满足前置即阻断”。
- 回滚：
  - 立即回退到 `dry_run/recon` 默认模式。

## 风险 2：读写域分离未落地导致跨域误放行

- 等级：high
- 触发条件：读域结论被用于写域放行，或目标域未显式确认。
- 影响：写域风险在无充分前置下被放大。
- 缓解：
  - 强制 `target_domain` 与 `target_page` 显式确认。
  - 禁止单域成功推导另一域放行。
- 回滚：
  - 触发即暂停该 profile 的 live 升级能力。

## 风险 3：审批与审计字段不足，无法追溯放行责任

- 等级：high
- 触发条件：live 放行无 `approver/approved_at/checks` 或不可检索。
- 影响：无法复盘与问责，回滚动作失焦。
- 缓解：
  - 冻结审批记录最小字段并纳入测试。
  - 每次门禁判定都写审计记录。
- 回滚：
  - 视作审批无效，统一降级为 `dry_run/recon`。

## Stop-Ship 条件

- 未通过 FR-0010 spec review 即恢复 `#208` live 正式验证。
- 在无审批记录情况下放行 `live_read_high_risk` 或 `live_write`。

## 回滚策略

1. 立即停止高风险 live。
2. 门禁状态统一回退为 `paused`。
3. 清点最近放行记录并标记为待复核。
4. 重新进入 FR-0010 评审链路后再评估恢复。
