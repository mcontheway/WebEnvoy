# FR-0002 风险

FR-0002 定义的是 WebEnvoy 浏览器内执行链路的最小通信闭环，因此需要单独识别 link-layer 风险、断连风险和协议漂移风险。

## 关键风险

| 风险 | 触发条件 | 影响 | 缓解手段 | 观察信号 | 回滚 / 收口方式 |
|---|---|---|---|---|---|
| MV3 Service Worker 休眠导致静默断开 | Background 空闲后被 Chrome 休眠 | CLI 误以为链路仍可用，转发请求悬挂 | 强制心跳与断连判定；把 `disconnected` 做成显式状态 | 心跳缺失、连接对象失活、请求无回包 | 立即标记会话断开，要求重新握手 |
| 协议漂移 | 后续实现偷偷引入第二套 transport envelope | CLI、Background、Content Script 互相无法对齐 | 冻结 `contracts/native-messaging-bridge.md`；只允许追加字段，不允许改壳 | 新实现中出现不同的 `method` / `params` / `session_id` 约定 | 阻断合并，统一回到 FR-0002 契约 |
| 把页面业务错误误判成 transport 错误 | Content Script 返回业务失败但被 Background 重分类 | 上层无法区分“链路坏了”和“业务没成” | 规定 payload 只负责业务错误，transport 只负责链路错误 | 业务失败统一被报成 `ERR_TRANSPORT_*` | 回退到错误分类边界重新拆分 |
| 转发路径绕过 Content Script | 为了快，把消息直接从 Background 解释完再回 CLI | 失去页面上下文，后续 FR 无法承接 | 明确最小往返必须经过 Content Script | 测试中看不到页面侧收到消息 | 阻断合并，补回页面侧转发链路 |
| 多 tab 路由歧义 | 当前页面未绑定或同时存在多个候选 tab | 消息发错目标，结果不可复现 | 本 FR 只允许单一最小目标，不做 fan-out | 同一命令偶发落到不同页面 | 收口到单目标路由，禁止扩展到广播 |
| 超时判定缺失 | 只看最终错误，不看时间边界 | 调用方无法稳定做重试/恢复 | 冻结最小超时错误码与会话状态转移 | 请求长时间卡住而无错误 | 强制超时失败返回 |

## Stop-Ship 条件

出现以下任一情况时，不应进入实现或合并：

- 不能稳定区分握手失败、断连、超时和转发失败
- 不能保证 `runtime.ping` 的最小往返经过 Content Script
- 心跳缺失后仍然把会话当成可用
- 为了接入后续事项而提前扩张成完整消息总线

## 剩余风险

- Chrome / MV3 的行为细节可能在不同版本和浏览器分支上表现不同，正式实现前仍需用本地验证收口
- 断连后重连的具体退避时长可以在实现阶段收口，但不得影响本契约的状态语义
