# FR-0009 风险记录

## 风险清单

### R1：继续高风险 live 实验导致账号处罚升级

- 等级：high
- 触发条件：在风险预警后继续扩大 live 读写实验面
- 影响：功能受限、账号封禁、阶段验证失真
- 缓解：
  - 默认暂停高风险 live
  - 人工确认门禁
  - 仅允许 dry-run/侦察作为默认执行模式

### R2：读域/写域边界混淆导致错误放行

- 等级：high
- 触发条件：将 `www.xiaohongshu.com` 与 `creator.xiaohongshu.com` 视作同一执行边界
- 影响：登录态误判、注入范围越界、风险扩大
- 缓解：
  - 在契约中强制分离域名与能力范围
  - 禁止单域成功推断另一域放行

### R3：门禁结论不可消费导致后续事项继续漂移

- 等级：medium
- 触发条件：输出没有稳定对象，`#208`/后续事项无法引用
- 影响：重复争论、范围蔓延、阶段门不清晰
- 缓解：
  - 提供四对象稳定契约
  - 在 TODO 中明确消费关系与前置动作

### R4：人工确认责任与审批留痕边界不清导致误放行

- 等级：high
- 触发条件：恢复 limited / allowed live 时，没有明确责任人，或未来执行契约中的审批 / 审计载体仍不清晰
- 影响：无法复核放行依据、问题责任失焦、后续事项绕过统一口径
- 缓解：
  - 规约阶段默认由发起 live 恢复 / 扩展请求的实现负责人承担确认责任
  - formal closeout 留痕与未来执行层审批 / 审计载体必须分开描述
  - 未来 live 放行所需的正式审批 / 审计记录由 `FR-0010/0011` 承接

## Stop-Ship 条件

- 未通过 spec review 即试图恢复 `#208` 后续事项的高风险 live 验证。
- 未完成人工确认门禁设计即放行高风险 live。
- 缺少责任人，或未来执行契约中的审批 / 审计载体仍不可复核，即试图放行 limited / allowed live。


## 回滚策略

- 若后续实施偏离本 FR：
  - 立即恢复“暂停高风险 live”默认状态
  - 回退放行结论到 dry-run/侦察模式
  - 重新进入 FR-0009 审查链路后再评估恢复条件
