# FR-0013 Layer 2 事件级拟人模拟增强

## 背景

`FR-0011` 已为 Sprint 3 冻结最小反风控执行能力，重点是门禁主落点、最小风险状态机、读路径模式收敛，以及写路径的最小交互分级与阻断边界。它解决的是“哪些动作可以在什么风险状态下进入最小 live”，而不是“进入 live 后，事件层如何尽量接近真实用户”。

当前 Layer 2 owning Work Item 对应的缺口正处在这之后：`docs/dev/architecture/anti-detection.md` 已写出 Layer 2 事件级拟人模拟的方向，但当前 backlog 仍主要围绕“是否放行”而非“放行后的执行是否足够真实”。如果继续只沿用 Sprint 3 最小边界，后续 Phase 2 的读写封装、未知站点探索与页面预热会缺少统一的事件层增强策略。

因此，本 FR 作为 Phase 2 主树中的正式 FR 之一，负责把 Layer 2 的事件级拟人模拟增强收敛成可供后续实现 PR 消费的正式规约输入；对应的 owning Work Item 为 Layer 2 scope。

## 目标

1. 冻结 Layer 2 事件级拟人模拟增强的正式范围，明确其在 Phase 2 -> FR-0013 主树中的定位。
2. 冻结“真实交互优先、合成交互回退”的更细粒度策略对象，避免后续实现各写一套判断。
3. 冻结 `focus/input/change/blur` 等事件链一致性要求，以及滚动、悬停、输入节奏等最小拟人化边界。
4. 冻结 Layer 2 与读路径、写路径调用链的衔接方式，使后续实现可在不改写 FR-0011 状态机的前提下接入。
5. 输出 implementation-ready 的规约输入，供后续实现 PR 使用。

## 非目标

- 不重定义 `FR-0011` 已冻结的风险状态机、审批对象、审计对象或写路径动作分级。
- 不交付完整 Layer 3 session 节律引擎。
- 不交付 Layer 4 平台长期画像或平台行为模型。
- 不交付小红书完整发布闭环。
- 不恢复 `#208` live 正式验证，也不把本 FR 当作 `#208` 的关闭依据。
- 不与 `#227`、`#208` 或任何实现代码混在同一个 PR 中。

## 功能需求

### 1. Phase 2 定位与 FR-0011 边界冻结

- 必须明确本 FR 属于 Phase 2 主树中的 Layer 2 反风控延续建设，对应 owning Work Item 为 Layer 2 scope。
- 必须明确 `FR-0011` 是本 FR 的最小前置：
  - `FR-0011` 负责冻结最小风险门禁、最小写路径分级与阻断边界。
  - 本 FR 负责在这些前置已存在的前提下，补齐更完整的 Layer 2 事件级拟人模拟增强。
- 必须明确本 FR 不得重定义 `FR-0011` 的 `risk_state_machine`、`issue_action_matrix`、`write_interaction_tier`、`session_rhythm_policy`、`approval_record`、`audit_record`。

### 2. 真实交互优先与合成交互回退策略

- 必须定义稳定的事件策略对象，至少覆盖：
  - `click`
  - `focus`
  - `keyboard_input`
  - `composition_input`
  - `scroll`
  - `hover`
- 每类动作都必须显式声明：
  - 首选路径是否为真实物理输入（CDP `Input` 或 OS 级）
  - 何时允许回退到 JS 合成事件链
  - 回退前置与阻断条件
  - 回退后的等待与观测要求
- 必须明确：
  - 普通点击与普通键入默认优先真实交互
  - 富文本/输入法相关场景允许“真实 focus + 合成输入链”混合路径
  - 回退策略不得绕过 `FR-0011` 的动作分级与风险状态约束

### 3. 事件链一致性

- 必须定义稳定的事件链策略对象，至少覆盖以下链路：
  - `focus` 获取链
  - 普通文本输入链
  - 富文本/输入法输入链
  - `change/blur` 收口链
  - 悬停后点击链
- 必须明确事件链的最小一致性要求：
  - 不能只注入单个 `input` 事件就视为完成
  - React/Vue 受控组件必须补齐 `change`
  - 需要失焦提交或校验的场景必须补齐 `blur`
  - 事件执行后必须进入状态收敛等待
- 必须明确“允许裁剪的链路”与“不可缺失的关键事件”，避免实现阶段随意省略。

### 4. 最小拟人化节奏与事件组合

- 必须定义 Layer 2 的最小节奏对象，至少覆盖：
  - 悬停确认窗口
  - 点击落点随机化
  - 输入字符间延迟与标点停顿
  - 偶发长停顿
  - 滚动分段、停顿与回头翻看
- 必须明确这是事件级边界，而不是 Layer 3 session 行为引擎。
- 必须明确最小拟人化边界只要求“消除明显机械感”，不要求生成长期 persona 或整 session 的高阶行为画像。

### 5. 与读路径 / 写路径调用链衔接

- 必须明确 Layer 2 增强接入的调用边界：
  - 读路径中的页面预热、悬停、滚动与可逆交互
  - 写路径中的 focus、点击、输入、失焦与操作后等待
- 必须明确 Layer 2 的选择结果如何被调用方消费：
  - 调用方先经过 `FR-0011` 门禁与动作等级判定
  - 通过后再选择 Layer 2 事件策略
  - 事件执行结束后走统一状态收敛等待与结构化结果回传
- 必须明确：
  - Layer 2 是执行增强层，不直接改写上层命令契约
  - L2/L3 读写实现都可以消费同一套事件策略对象，但平台专用适配器可覆盖默认值
  - Layer 2 策略选择只发生在当前已选定的执行路径内，不负责运行时自动切换 L3/L2/L1

### 6. 观察、诊断与失败回退

- 必须定义 Layer 2 执行时最小需要回传的结构化字段，至少覆盖：
  - 选择的交互路径（真实/合成/混合）
  - 触发的事件链名称
  - 节奏配置来源（默认/平台覆盖）
  - 回退原因
  - 状态收敛结果
- 必须明确事件链失败的最小分类：
  - 焦点未建立
  - 事件链未触发框架状态更新
  - 页面结构/交互目标漂移
  - 被 `FR-0011` 风险边界阻断
- 必须明确失败时只能在已允许的策略内回退，不得自动升级到更高风险动作。

## GWT 验收场景

### 场景 1：FR-0011 与 FR-0013 边界清楚

Given `FR-0011` 已冻结最小门禁、状态机与写路径交互分级
When reviewer 检查 FR-0013 套件
Then 能明确看到 FR-0013 只承接事件级拟人模拟增强
And 不会重定义 `FR-0011` 的状态机、审批或审计对象

### 场景 2：真实交互优先与合成交互回退被对象化

Given 调用方已通过 `FR-0011` 的动作分级与风险状态检查
When 后续实现为点击、普通输入或富文本输入选择事件策略
Then 能从规约中直接得到首选路径、允许回退条件与阻断条件
And 富文本场景会明确使用“真实 focus + 合成输入链”的混合路径边界

### 场景 3：事件链一致性可被实现与测试直接消费

Given 页面交互需要触发前端框架状态更新
When reviewer 检查事件链策略对象
Then 能明确看到 `focus/input/change/blur` 等关键事件的最小要求
And 不存在“只发一个 input 事件就算完成”的模糊空间

### 场景 4：最小拟人化边界不会膨胀成 Layer 3

Given Layer 2 需要降低明显机械感
When reviewer 检查节奏与滚动策略
Then 能看到悬停、输入延迟、滚动分段与回头翻看的最小要求
And 文档会明确这些要求不等于完整 session 行为引擎

### 场景 5：读写调用链可以在不改写上层契约的前提下接入

Given 读路径和写路径都可能消费 Layer 2 事件增强
When reviewer 检查调用边界
Then 能明确看到“先走 FR-0011 门禁，再选 Layer 2 策略，再做状态收敛等待”的顺序
And 不需要新增另一套 CLI 或风险门禁契约

## 异常与边界场景

1. 页面只接受真实输入：
   - 若目标组件对 `isTrusted` 或真实 focus 有硬要求，合成路径只能作为受限回退或直接阻断，不得默认为成功替代。
2. 富文本/输入法链与普通输入链混淆：
   - 普通输入、富文本输入、上传注入必须分开定义，不得把富文本混合链推广为所有输入默认值。
3. 交互节奏被误写成 session 行为引擎：
   - 若规约试图引入长期 persona、任务后闲逛、跨页面节律编排，应视为越界到 Layer 3。
4. 回退路径绕过门禁：
   - 若实现通过“回退到合成事件”绕开 `FR-0011` 已阻断的动作等级，视为阻断性违规。
5. 读路径页面预热膨胀成正式 live 恢复：
   - 页面预热、滚动、悬停只定义事件增强边界，不等于恢复 `#208` live 正式验证。
6. 平台专用需求挤占通用边界：
   - 小红书特定发布链、平台长期画像或定制 session 行为，不得写进本 FR 的通用 Layer 2 基线。

## 验收标准

1. FR-0013 套件完整，至少包含 `spec.md`、`plan.md`、`TODO.md`、`research.md`、`risks.md`、`data-model.md`，并按需创建 `contracts/`。
2. Phase 2 / `FR-0013` / Layer 2 owning Work Item / `FR-0011` 的边界与依赖关系已明确冻结。
3. 真实交互优先、合成交互回退、混合路径、事件链一致性与最小拟人化节奏均已形成稳定规约对象。
4. 读路径/写路径调用链的接入顺序与状态收敛要求已明确，后续实现无需再自造新边界。
5. 文档明确不承诺 Layer 3、Layer 4、小红书完整发布闭环或 `#208` live 恢复。
6. 本 FR 保持为规约审查输入，不混入实现代码。
