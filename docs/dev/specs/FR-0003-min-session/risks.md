# FR-0003 风险

## 风险 1：Profile 独占锁失效导致并发写入

- 触发条件：同一 Profile 被两个 CLI 进程同时启动或登录
- 影响：UserDataDir 与元数据被交叉写入，导致会话损坏
- 缓解：启动前强制获取锁，锁冲突直接拒绝；锁中记录 ownerPid、ownerRunId 和心跳
- 回滚 / 降级：释放可识别的陈旧锁后再重试，不做静默并发

## 风险 2：代理静默改绑导致会话出口漂移

- 触发条件：同一 Profile 再次启动时传入不同代理，或代理参数解析异常
- 影响：会话出口变化，登录态和风控画像失去一致性
- 缓解：代理绑定写入 ProfileMeta 后默认复用；出现冲突直接报错，不自动轮换
- 回滚 / 降级：显式清除绑定后再重新建立，不在后台偷偷替换

## 风险 3：元数据写入失败或文件损坏

- 触发条件：进程中断、磁盘异常、权限不足、手工篡改
- 影响：状态回读错误，Profile 无法恢复到正确状态
- 缓解：元数据采用原子写入和最小字段集；读失败时优先报告可诊断错误
- 回滚 / 降级：保留最近一次成功元数据，必要时只读启动并禁止状态变更

## 风险 4：浏览器启动成功但运行时未真正可用

- 触发条件：Chrome 拉起了进程，但 Profile 尚未稳定就绪或窗口异常关闭
- 影响：上层命令误以为会话可用
- 缓解：`runtime.start` 只有在状态进入 `ready` 后才算成功；中间态必须可追踪
- 回滚 / 降级：标记为 `disconnected`，由用户重新启动，不做自动掩盖

## 风险 5：手动登录流程被误判为自动完成

- 触发条件：登录页跳转、验证码、风控弹窗或用户中断
- 影响：`runtime.login` 将不完整状态误写成成功
- 缓解：登录确认必须有明确的人类确认或可验证完成信号
- 回滚 / 降级：保持在 `logging_in` 或回到 `disconnected`，不向 `ready` 伪造回写

## 风险 6：陈旧锁长期阻塞同一 Profile

- 触发条件：CLI 崩溃、断电或外部强杀导致锁未释放
- 影响：Profile 被永久占用，无法恢复
- 缓解：锁记录 ownerPid、ownerRunId 与最后心跳；重启时识别陈旧锁并允许回收
- 回滚 / 降级：回收陈旧锁后重新进入 `starting`，但必须保留审计痕迹

## 风险 7：最小恢复字段泄漏或膨胀

- 触发条件：`fingerprintSeeds` 或 `localStorageSnapshots` 被当作通用配置继续扩写，或被错误输出到日志 / 诊断面
- 影响：Profile 恢复信息外泄，元数据逐渐膨胀为账号资产总表
- 缓解：限定字段只服务最小会话恢复；日志与诊断默认脱敏，不输出原始快照内容
- 回滚 / 降级：一旦发现结构外溢，优先截断字段并回收为只保留最小恢复所需键值
