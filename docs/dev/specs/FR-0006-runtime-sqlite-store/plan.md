# FR-0006 实施计划

## 实施目标

本 FR 的实现只交付 SQLite 最小运行记录基座，形成可查询、可审计、可复用的运行证据层。

本次实现完成后应交付：

- 最小 schema（运行主记录 + 运行事件记录）
- 写入层最小能力（初始化、幂等主记录写入、事件追加写入）
- 查询层最小能力（按 `run_id` 查询主记录与时间线事件）
- 与 `#154` 对齐的最小诊断字段落库能力

本次不交付：

- 完整业务数据仓库
- 复杂迁移编排系统
- 实时会话状态机真相源

## 分阶段拆分

### 阶段 A：冻结数据模型与边界

产出：

- `data-model.md` 冻结最小实体、字段与约束
- `contracts/runtime-store.md` 冻结写入与查询契约
- 明确与 `#143/#154/#159` 的边界

依赖：

- FR-0001、FR-0003、FR-0004 已有 spec 可引用

### 阶段 B：实现最小 schema 与初始化

产出：

- SQLite 初始化流程
- schema 版本标识与最小迁移入口（仅当前版本）
- WAL 模式启用与启动校验

依赖：

- 阶段 A 完成

### 阶段 C：实现写入与查询能力

产出：

- 主记录幂等写入
- 事件追加写入
- 按 `run_id` 读取主记录与事件时间线

依赖：

- 阶段 B 完成
- `#143` 的 `run_id/session_id/profile` 字段稳定可引用

### 阶段 D：接入诊断映射与收口验证

产出：

- 与 `#154` 对齐的最小诊断字段映射
- 契约测试、边界测试与最小人工验证证据

依赖：

- 阶段 C 完成
- `#154` 的诊断字段定义冻结

最小闭环：

- 到阶段 C 结束，必须已经可以按 `run_id` 查询一次运行的主记录与关键事件。

## 实现约束

- SQLite 不能成为实时会话状态真相源，`runtime.status` 仍由 `#143` 路径输出。
- 不落敏感原文：Cookie、Authorization、Token、完整 query 参数、完整响应体禁止入库。
- 不改变 FR-0001 的 CLI 外层响应壳和退出码语义。
- 不引入平台业务详情表、全文索引或媒体存储。
- 不在本 FR 内扩张到能力版本仓库或发布系统。

## 测试与验证策略

单元测试：

- schema 初始化与版本检查
- 主记录幂等写入逻辑
- 事件写入关联约束
- 脱敏与截断规则

契约测试：

- `contracts/runtime-store.md` 的输入输出与错误码
- `run_id` 唯一性约束与查询结构稳定性
- 主记录存在前提下事件追加行为

集成测试：

- 成功路径：命令成功后可查询主记录与至少两条关键事件
- 失败路径：命令失败后仍可查询失败主记录与失败事件
- 锁冲突路径：短时冲突重试与超限错误返回

人工验证：

- 在本地运行一次 CLI 命令，确认可通过 `run_id` 追溯最小时间线
- 验证数据库中无敏感字段明文

完成证据：

- 单元、契约、集成测试通过
- `bash scripts/docs-guard.sh` 通过
- `bash scripts/spec-guard.sh` 通过

## TDD 范围

默认先写测试的模块：

- schema 初始化与版本校验
- 主记录幂等写入
- 事件关联约束
- 脱敏与截断逻辑
- 读模型序列化

暂不强制先写测试的部分：

- CLI 命令端到端人工 smoke
- 极端性能压测

划分理由：

- 前者属于稳定且确定性的共享逻辑，后续 `#154/#159` 会复用，必须先测后码。
- 后者依赖运行环境和时序，不适合作为本 FR 的首轮阻断门槛。

## 并行 / 串行关系

前绪阻塞：

- `#141` 已冻结 CLI 契约
- `#143` 已提供稳定的 `run_id/session_id/profile` 运行标识

可并行项：

- 本 FR 的 schema/写入契约设计可与 `#154` 的诊断字段冻结并行推进
- 本 FR 的运行记录查询契约可与 `#159` 的能力壳字段映射并行对齐

串行关系：

- 先冻结 `data-model/contracts`，再进入实现
- 先完成 schema 初始化，再接写入与查询
- 先完成写入正确性验证，再开放后续 FR 消费

## 进入实现前条件

只有满足以下条件，才允许进入 `feat/FR-0006-*` 实现分支：

- 当前 spec-only 变更已在 Draft PR 或等价评审上下文中
- spec review 结论为 `APPROVE`
- `ready_for_implementation = true`
- `data-model.md`、`contracts/runtime-store.md`、`risks.md` 已随套件一起评审通过
- `TODO.md` 中的 review blockers 已清空

在此之前，禁止：

- 修改运行时代码或数据库实现代码
- 将 SQLite 扩张为实时状态真相源
- 将本 FR 与实现代码混在同一 PR
