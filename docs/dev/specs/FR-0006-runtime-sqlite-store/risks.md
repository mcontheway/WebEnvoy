# FR-0006 风险与回滚

## 风险 1：SQLite 被误用为实时会话真相源

触发条件：

- 在 `runtime.status` 或会话恢复逻辑中直接读取运行记录来判定当前活跃状态。

影响：

- 与 `#143` 的 profile/session 状态机冲突，出现“历史记录覆盖实时状态”的错误判断。

缓解：

- 在契约与实现中明确：SQLite 仅为历史证据层。
- `runtime.status` 路径禁止依赖运行记录表作为实时判定输入。

观察信号：

- 状态查询逻辑出现对 `runtime_runs` 的直接依赖。

回滚 / 降级：

- 回滚到仅写入运行记录、不参与实时状态判断的版本。

## 风险 2：敏感数据落库导致泄露面扩张

触发条件：

- 直接写入完整请求体、完整 URL query、Cookie、Authorization 或 Token。

影响：

- 调试库变成高敏感数据存储点，扩大安全和合规风险。

缓解：

- 写库前统一执行脱敏与截断。
- 契约层禁止上述字段入库。

观察信号：

- 抽样记录出现敏感关键字或超长原文载荷。

回滚 / 降级：

- 立即停用高风险字段写入，执行数据清理脚本移除已落敏感记录。

## 风险 3：并发写入冲突造成记录缺失或重复

触发条件：

- 同一 `run_id` 被并发 upsert，或事件写入与主记录写入时序冲突。

影响：

- 主记录重复、孤儿事件或关键事件丢失，破坏可审计性。

缓解：

- 主记录唯一键 + 幂等 upsert。
- 事件写入前校验主记录存在。
- 冲突采用有限重试，超限返回可重试错误。

观察信号：

- 同一 `run_id` 出现多条主记录或孤儿事件。

回滚 / 降级：

- 切回串行写入路径并暂停并发优化。

## 风险 4：schema 变更失败导致运行写入中断

触发条件：

- 部署版本与本地数据库 schema 版本不兼容。

影响：

- 运行记录写入失败，CLI 缺少排障证据。

缓解：

- 启动时执行 schema 版本检查。
- 不兼容时返回明确错误码并阻断写入，避免静默损坏。

观察信号：

- 出现 `ERR_RUNTIME_STORE_SCHEMA_MISMATCH`。

回滚 / 降级：

- 回滚到上一兼容 schema 版本，或执行受控迁移后再恢复写入。

## 风险 5：记录体积膨胀导致性能退化

触发条件：

- 事件摘要无限增长或单次命令写入事件过多。

影响：

- 写入延迟上升，查询时间线退化，影响 CLI 响应稳定性。

缓解：

- 摘要字段长度上限与截断标记。
- 关键事件最小集策略，避免无价值噪声事件。

观察信号：

- 数据库体积增长异常，单次命令事件条数持续超阈值。

回滚 / 降级：

- 降级为仅保留主记录 + 失败关键事件，暂停非关键事件写入。
