# FR-0008 实施计划

## 实施目标

在不编写平台实现代码的前提下，交付一套可评审、可复用的 Spike A-Write 正式规约，明确小红书最小写链路的富文本输入、图片上传和最小交互动作候选的证据边界，并把结论约束成 `#208` 与后续写能力 FR 可直接消费的输入基线，而不是直接宣布 Phase 1 blocker 已完成。

## 分阶段拆分

### 阶段 1：Spike charter 与样本矩阵冻结

- 产出：决策问题、时间盒、未知项、候选动作清单
- 依赖：FR-0002/0003/0004/0007 已形成最小运行与观测承载面

### 阶段 2：写链路证据采样与分类

- 产出：富文本编辑器证据矩阵、图片上传证据矩阵、候选最小交互动作列表
- 依赖：阶段 1 的场景矩阵；同类页面多轮观测结果

### 阶段 3：双门禁与 handoff 契约化

- 产出：`contracts/` 下的结构化输出边界、`research.md` 中的证据矩阵、`risks.md`
- 依赖：阶段 2 结论已按 `primary/fallback`、`page/api` 收敛

### 阶段 4：规约评审收口

- 产出：可进入 spec review 的 `spec.md`、`plan.md`、`TODO.md`
- 依赖：阶段 3 完成，且 `#146` 与 `#208` 边界不再歧义

## 实现约束

1. 本 FR 仅承担侦察输入与规约冻结，不实现页面写入、上传或发布命令。
2. 不把 `#146` 作为 Phase 1 阶段出站项；`#208` 仍保留正式验证职责。
3. 不新增或修改数据库、会话 schema 或公共运行时接口。
4. 不把单轮页面实验、仓库外截图或不可复核环境当成正式结论。
5. 页面级主路径失败时，必须保留 API fallback 或 candidate 证据；不得为了看起来闭环而删除失败样本。
6. 只有达到多轮复现、风险边界清楚、前后置信号稳定的证据，才允许标为 `admission_ready`。
7. fallback 证据只用于风险收口、诊断连续性或 `#208` 候选输入，不得直接放行实现。
8. 涉及账号、上传、发布、不可逆提交的风险必须写入 `risks.md`，不得口头略过。

## 测试与验证策略

1. 文档门禁：
   - `bash scripts/docs-guard.sh`
   - `bash scripts/spec-guard.sh`
2. 侦察结论验证：
   - 富文本编辑器至少验证入口、焦点、输入事件链、失败回显四类信号
   - 图片上传至少验证入口类型、文件注入方式、页面上传反馈、失败反馈四类信号
   - `#208` handoff 中的候选动作至少包含一轮最小复现实验描述
3. 评审验证：
   - 使用 `spec_review.md` 复核“侦察输入 vs 阶段出口”边界
   - 对照 `docs/dev/roadmap.md` 确认未把完整写链路提前膨胀进 Phase 1 blocker

## TDD 范围

1. 本 FR 本身为规约与侦察，不要求新增实现代码测试。
2. 先为后续实现冻结 TDD 要求：
   - 富文本事件编排与状态收敛等待：必须 TDD
   - 文件注入、上传节点识别与错误映射：必须 TDD
   - `#208` 选定动作的成功/失败判定器：必须 TDD
3. 人工侦察和页面观测不适用 TDD，但必须保留可复现实验步骤。

## 并行 / 串行关系

### 串行阻塞

- `#142`、`#143` 未达到可用状态前，浏览器内写链路侦察无法稳定执行。
- `#154` 未提供最小观察与错误归类时，写链路失败样本无法被正确冻结。
- FR-0008 spec review 未通过前，不进入小红书写能力实现，也不把 `#208` blocker 结论建立在本 FR 之上。

### 可并行

- 与 `#145` / FR-0005 可并行：读链路与写链路侦察分开管理，但共享会话与观察承载面。
- 与 `#208` 可并行准备：`#208` 可以先定义正式验证问题，但不得抢跑使用未冻结的 `#146` 结论。
- 与后续写能力实现 FR 的范围设计可并行草拟，但实现不得先于本 FR 的 spec review。

## 进入实现前条件

1. FR-0008 规约 PR 完成 spec review，并得到“仅作为侦察输入”或“可进入后续实现 FR 输入”结论。
2. `contracts/xhs-write-spike.md` 中四类核心对象（`editor_path`、`upload_path_catalog`、`minimal_action_candidates`、`gate_status`）的字段语义无歧义。
3. `#208` 若要消费本 FR，必须显式指出所选候选动作与未消费的剩余写链路范围。
4. `risks.md` 中账号、上传、不可逆提交风险均已有缓解与回滚路径。
5. 若当前只有 fallback-only 或 `candidate` 证据，后续实现 FR 必须继续补主路径复核，不得据此直接开发。
