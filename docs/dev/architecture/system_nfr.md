# 系统非功能性需求 (Non-Functional Requirements)

> 本文档汇总 WebEnvoy 的可量化非功能性需求指标，作为各层文档中散落指标的唯一汇聚点。
> 如与其他文档中的指标存在冲突，**以本文档为准**。

---

## 一、成功率指标（QoS）

| 执行层级 | 场景 | 目标成功率 | 说明 |
|---|---|---|---|
| **L3 专用引擎** | 已知平台，全自动环境 | ≥ 95% | Phase 1 核心验收指标 |
| **L2 通用读取** | 未知网站，AX Tree 可解析 | ≥ 80% | Phase 2 核心验收指标 |
| **L1 VLM 漫游** | L2 失效的边缘站点 | ~60% | 兜底，不作为主要 KPI |

## 二、性能指标

| 指标 | 目标值 | 说明 |
|---|---|---|
| L3 API 读取响应时间 | 毫秒级 | Content Script fetch() 到 CLI 收到摘要 |
| L2 通用读取响应时间 | ≤ 2s | 启用请求拦截（屏蔽图片/广告/字体）后的页面加载+解析总时长 |
| Native Messaging 单条消息大小 | ≤ 1MB | Chrome 原生限制，超出时自动分片（见 system-design.md §10.3） |
| CLI 进程启动时间（含 Extension 建立连接）| ≤ 5s | 包含 Playwright 启动、Profile 加载、Native Messaging 握手 |

## 三、稳定性指标

| 指标 | 目标值 | 说明 |
|---|---|---|
| 批量任务断点续传覆盖率 | 100% | 所有批量任务必须在 SQLite 中记录进度快照（RCV-01）|
| 通信断开自动重连时间 | ≤ 30s | 超时后提示 AI 人工介入（ERR-07） |
| 断线期间命令排队上限 | 5 条 | 超出后丢弃并通知 AI（ERR-07）|

## 四、执行身份与风控安全指标

| 指标 | 目标值 | 说明 |
|---|---|---|
| 配置空间并发冲突率 | 0 | 同一 Profile 同时只允许一个 CLI 进程持有，避免 Cookie / LocalStorage 冲突 |
| 已登录配置空间重启后可恢复率 | 稳定复用直到登录态过期 | Cookie 与 LocalStorage 恢复属于当前最小会话能力的一部分 |
| 指纹指标稳定性（同 Profile 跨 Session）| 无随机漂移 | 噪声种子固化，Canvas/Audio 哈希在同一环境下必须稳定 |
| 高风险平台 7 日封禁率 | 不高于同批人工基线 | 仅作为 Phase 4 及后续扩展的验收指标；必须用真实平台 A/B 测试验证，若未建立人工基线，不得宣称通过该项验收 |
| 高风险平台 7 日限流 / 强验证触发率 | 不高于同批人工基线 | 仅作为 Phase 4 及后续扩展的验收指标；与封禁率一起作为风控抑制效果的核心结果指标，不能仅用本地指纹稳定性替代 |

## 五、平台适配维护指标

| 指标 | 目标值 | 说明 |
|---|---|---|
| 新平台 API 端点"探测→验证→固化"周期 | ≤ 10 分钟 | 通过 AX Tree 语义查询 + `recon` 工具链实现（ARCHITECTURE_PRINCIPLES.md §6）|
| 规则热更新生效延迟 | 0s（下次操作立即生效）| chokidar 文件监听，无需重启 CLI（system-design.md §8.4）|
| L3 Adapter 失效检测响应 | 连续 3 次异常后返回明确错误 | 包含降级建议和诊断信息（system-design.md §11.2）|

## 六、资源消耗约束

| 资源 | 约束 | 说明 |
|---|---|---|
| SQLite 数据库写并发 | 串行（WAL 模式保证）| 同一 Profile 同时只允许一个 CLI 进程操作（SCC-07）|
| 同一 Profile 并发进程数 | = 1 | 配置空间独占锁（lock 文件），防止 Cookie 冲突（SCC-07）|
| 媒体文件 Blob 存储 | 不入 SQLite | 只存文件路径引用，Blob 落磁盘（system-design.md §12）|
