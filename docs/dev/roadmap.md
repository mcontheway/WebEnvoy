## Phase 2：通用读取底座与适配器固化闭环

> 目标：在 Playwright 之上构建 **L2 通用读取层**，使 WebEnvoy 能够对任意未知网站提供可靠的内容提取与交互能力（这是真正的通用能力，而非仅覆盖已知平台）。L1 VLM 漫游作为最终兜底。在 L2 底座稳定后，实现从漫游路径到 L3 适配器的自动固化闭环。
>
> **关键区分**：L2 通用层 ≠ bb-sites 适配器集合——bb-sites 提供的是 35 个已知平台的 API 捷径（属于 L3），无法覆盖任意未知网站，L2 才是真正的通用能力。
>
> 指标：L2 层对未知网站首次读取成功率 ≥ 80%；启用请求拦截优化后任意 URL 内容提取响应时间 ≤ 2s；"探测→验证→人工微调→固化"的单点流转周期 ≤ 10 分钟。

**[L2 通用读取层]**
> 基于 Playwright 构建 AI 友好的通用读取原语，核心是 AX Tree 压缩与 RefMap 元素引用系统。移植 agent-browser 的 `processAriaTree` + `compactTree` + RefMap 短引用算法；实现通用 CLI 原语 `navigate` / `snapshot` / `click` / `type` / `intercept` / `extract`；集成 Steel Browser 请求拦截策略屏蔽图片/广告/字体资源（加载提速 5-10x）；支持正文 Markdown 提取（NAV-04）与登录墙/验证码检测（NAV-08 / ERR-01）。

**[L1 VLM 漫游兜底]**
> 截图 + 坐标 + 物理点击，覆盖 AX Tree 无法解析（渲染异常、强 Canvas 化）的边缘站点。

**[适配器固化流水线]**
> 将 L2 漫游路径的探索结果提炼为可复用的 L3 适配器规则，跑通"探测→验证→人工微调→固化"闭环。

**[L3 平台横向扩张（可选）]**
> 视 Phase 1 后的资源情况，按需扩展更多平台的 L3 专用适配器。属于对已知平台的速度优化，不是通用能力。
