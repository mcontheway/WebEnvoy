# WebEnvoy 愿景 (Vision)

---

## 一、 一句话 (Elevator Pitch)

**WebEnvoy 是专为 AI Agent 打造的高速、高隐匿 Web 执行工具（Tool）**，让 Agent 像调用本地函数一样稳定穿透高防平台的风控，完成结构化数据抓取与复杂交互——而不是靠视觉截图低效漫游页面。

## 二、 问题陈述 (The Problem)

当前的 AI Agent（如基于浏览器/多模态视觉的自动化方案）在面对真实商业 Web 环境的攻防时，显得极其脆弱、低效且昂贵：

1.  **速度与成本的灾难**：让大模型通过“获取无障碍树/截图 -> 推理按钮位置 -> 等待页面加载 -> 再次验证”的闭环操作网页，速度极慢，且会海量吞噬宝贵的 Token 上下文窗口。
2.  **脆弱的系统稳定性**：强依赖目标前端的 UI 结构，一旦网页改版或元素位移，基于视觉坐标或硬编码选择器的脚本将立即崩溃。
3.  **极高的风控拦截率**：直接调用传统的无头浏览器（Headless Browser）或 CDP 原生协议，极易触发目标网站（如小红书、抖音、Cloudflare）的风控探针，被识别为机器人并无情封锁。

随着大语言模型能力的爆发，AI 智能体与真实互联网交互的**行动力（Action Capability）不足**已成为制约其落地的核心瓶颈。

## 三、 项目愿景 (The Vision)

WebEnvoy 的目标是将高防 Web 平台的复杂交互，收敛为 AI Agent 可直接调度的**结构化执行接口**，并将执行链路的完整控制权交还给上层智能体。

我们致力于实现：**AI 只需表达业务意图，WebEnvoy 负责择优执行并实时反馈链路细节。** 我们通过在任意平台上实现**高成功率、低风控风险、极低成本与极低上下文占用**的读写闭环，赋予大模型重塑互联网生态的行动力。

**我们的核心系统哲学：**

1.  **链路完全透明与深度介入（核心竞争力）**：拒绝黑盒化。WebEnvoy 开放网络层与 DOM 层的**侦察与调度原语**。允许主控 AI 像指挥特战小队一样，实时监控交互节点（如流量特征、JS 变量、风控探针），并在关键时刻执行微观干预（如修改请求指纹、拦截异常上报），而不是死等成功或失败的最终结果。
2.  **执行路径的自动择优与降级**：借鉴“站点适配器分层（Tiered Adapters）”模式，系统会自动在 API 直连（极速）、内存窃取（零渲染）与拟人化 UI 模拟（高稳健）之间动态切换。始终以满足意图所需的**最低代价**路径作为首选。
3.  **高防攻坚与工业级接入**：通过对头部高防平台的深度逆向，沉淀通用的抗检测工程底座（指纹混淆、行为建模）。同时提供配套的侦察工具链，将新平台的适配周期从"天"缩短至"分钟"，实现从"特供定制"到"规模化通用"的跨越。
4.  **开放的站点适配器生态**：WebEnvoy 核心保持精简，面向特定平台的执行逻辑以**独立站点适配器**的形式组织。任何开发者都可以贡献新的站点适配器，官方维护高频平台的官方适配器，社区负责长尾覆盖——形成一个由用户驱动持续进化的 Web 执行网络。

## 四、 核心原则与价值观 (Core Principles / Values)

当我们在架构设计或功能开发面临取舍时，应遵循以下最高原则：

1.  **效率优先**：以最小代价（Token 消耗、推断时间）换取最精确的结构化结果。抵制会滥用上下文的重型通信模式。
2.  **特定平台优先于通用覆盖**：先打通高价值平台（主流社媒）的专属逆向通道（L2/L3），保障 95%+ 的成功率；再以通用底层漫游（L1）兜底未知站点。不为形式上的“通用化”牺牲实战可用性。
3.  **链路完全可控**：系统不是黑盒。主控 AI 或开发者必须能在链路各节点（网络层、DOM 层）介入侦察和调度。
4.  **账号安全与痕迹消除**：对于高风险平台，必须尽可能保障用户账号安全，深度抹除自动化痕迹（指纹特征、行为规律等），将降低风控风险视为与任务成功率同等重要的核心指标。
5.  **代码资产 100% 自主**：可借鉴开源竞品的攻防思路与业务数据字典，但核心机制必须用自主技术栈重写，确保合规与独立。

## 五、 目标受众 (Target Audience)

*   **主要用户**：构建 AI 智能体应用（AI Agents）的开发者、RPA 工程师、数据及业务增长黑客。他们急需一个稳定、不被风控阻断的底层交互管道来连接 LLM 和真实商业网站。
*   **次要用户**：具有跨平台内容自动化分发、竞品监测或高频操作需求的商业化运营团队。

## 六、 非目标 (Non-Goals / Out of Scope)

划定清晰的边界，防止需求无止境的“范围蔓延”（Scope Creep）。**以下内容我们坚决不做**：

1.  **我们不做自主决策的 Agent（核心底线）**：WebEnvoy 将自身的定位严格界定为供智能体调用的“外挂执行工具（Tool）”。所有的任务规划、意图理解判断、长篇图文润色等复杂逻辑推理都应交由外部的 AI 大脑去完成，我们只解决“如何最快、最稳地跨越网络风控并送回极简干净结构化数据”的苦差事。
2.  **我们不造大模型或聊天前端**：WebEnvoy 不提供面向终端大众的 Chatbot 交互界面。我们只做大语言模型背后的那双“执行系统的手”。
3.  **我们不盲目追求虚幻的“100% 零配置的通用化”**：面对高频高防护的头部阵地，我们承认并积极维护特定站点的适配方案，绝不为了追求形式上的通用而牺牲 95% 以上的实战可用性。

## 七、 成功标准与里程碑 (Success Criteria / Milestones)

1.  **Phase 1：特定业务极速闭环（实战验证）**
    *   里程碑：打通 2-3 个国内头部高防社媒平台（如小红书/抖音）的核心链路（读抓取 + 写图文混排/上传）。
    *   指标：在全自动化环境下的稳定操作成功率达到 95% 以上，初步建立极高的行业壁垒与口碑。
2.  **Phase 2：抽象出标准的 PlatformAdapter 接口（架构升维）**
    *   里程碑：完成统一平台适配器的事件总线抽象。
    *   指标：依靠内置的侦察与验证工具链，实现新平台核心接口的“分钟级”逆向，使适配器的开发与固化周期从“天”级缩短至“10分钟级”。
3.  **Phase 3：引入高防隐匿的兜底漫游容器（边界扩展）**
    *   里程碑：构建通用的 Fallback 漫游底座，集成指纹掩盖、自适应选取器等防检测技术。
    *   指标：在面对未知或改版严重的“荒野”系统时，L1 兜底初次执行成功率达到 60% 以上，并能自动反哺生成初步的适配器规则。
